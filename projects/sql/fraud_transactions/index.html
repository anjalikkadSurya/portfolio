Fraud Chain Detection

In this project, I analyzed a dataset of cryptocurrency transactions to identify potential fraudulent behavior based on suspicious chaining patterns.

  • Goal: Detect transaction chains occurring within 5 minutes of each other.
  • Criteria: Total amount > 150, with consecutive timestamps.
  • Output: Ranked chains per sender based on total loss.
View SQL Query

Expected Output

The result includes: sender_id, chain_id, chain_start, chain_end, total_amount, num_txns, fraud_rank, transaction_ids. It helps surface suspicious high-frequency, high-value transfer chains for further review.
